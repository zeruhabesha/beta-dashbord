Authentication Flow (Keycloak PKCE)

1. User hits /select
→
2. startKeycloakLogin()
→
3. Keycloak /authorize
  ?code_challenge=...
→
4. Code exchange
  → access_token
→
5. Token in
  sessionStorage
→
6. Bearer token
  on CMT calls
keycloak.js — Key Functions
  • getKeycloakConfig() — Read from window._env_ or import.meta.env
  • startKeycloakLogin() — OIDC auth code + PKCE redirect
  • initializeKeycloakSession() — Handle callback, exchange code
  • getKeycloakAccessToken() — Return current token
  • logoutFromKeycloak() — Logout + redirect
  • clearKeycloakSession() — Clear sessionStorage
accessControl.js — Key Functions
  • getConfiguredApiToken() — Read from localStorage
  • setConfiguredApiToken() — Store to localStorage
  • getConfiguredScopes() — Extract from token + storage
  • hasRequiredScopes() — Permission check
  • missingRequiredScopes() — Return what's missing
  • scopeMatches() — Wildcard scope matching (admin:*)

API Client Layer (src/api/)

opensearch.js
  • Direct OpenSearch cluster queries
  • GET /api/opensearch/_cat/indices
  • GET /api/opensearch/_search
  • GET /api/opensearch/{index}/_doc/{id}
opensearchDashboards.js
  • Data view (index pattern) management
  • Auto-repair missing saved objects
  • Called before iframe renders
  • Supports multi-title fallback
cmt.js — Case Management
  • Keycloak Bearer token authed
  • SSE live update support
  • Routes: /auth, /cases, /alerts, /report-templates, /users, /webhooks
  • Fallback: /api/v1/agents
siemAlerts.js
  • Real-time alert stream (SSE)
  • Case creation from triage
  • Alert acknowledgement
  • Enabled via VITE_SIEM_ALERTS_ENABLED
edr.js
  • EDR data from OpenSearch
  • Index: edr*, edr-detections-*, tenant-*-edr*
  • Detections, response actions, playbooks
  • Audit events, forensic data
securityServices.js
  • Playbook orchestration
  • Approval queue management
  • Threat hunting campaigns
  • Audit service log export
kubernetes.js
  • Namespace management for multi-tenancy
  • /api/kubernetes/api/v1/namespaces
  • mTLS for direct cluster access
  • Kind cluster auto-detection
otelApi.js
  • OpenTelemetry logs, traces, metrics
  • Index: opensearch_dashboards_sample_data_otel_*
  • Service map data
  • Distributed trace correlation
